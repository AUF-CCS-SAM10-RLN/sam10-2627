Linux CLI Reference

Linux Bash and CLI commands every systems administrator should know

Systems administration depends on command-line fluency. This page groups practical commands by daily admin tasks so students can build a usable working set.

Core commands

Purpose

Daily Ops

Inspection, troubleshooting, configuration, and recovery.

Navigation and files

Moving around and inspecting the filesystem

`pwd`

Show the current working directory.

`ls`, `ls -la`

List files, including hidden files and permissions.

`cd`

Change directories quickly and accurately.

`mkdir`, `rmdir`

Create and remove directories.

`cp`, `mv`, `rm`

Copy, move, and remove files and directories.

`touch`

Create an empty file or update timestamps.

`cat`, `less`, `head`, `tail`

Read full files or inspect the beginning and end of files.

`find`, `locate`

Search for files by name, path, or properties.

`grep`

Search text inside files and logs.

`wc`, `sort`, `cut`, `awk`, `sed`

Basic text processing for filtering, extracting, and transforming output.

Users and permissions

Identity, ownership, and access control commands

`whoami`, `id`

Check the current user and group membership.

`sudo`

Run administrative commands with elevated privilege.

`useradd`, `usermod`, `userdel`

Create, modify, and remove user accounts.

`groupadd`, `groupmod`, `groupdel`

Manage groups for access control.

`passwd`

Set or change account passwords.

`chmod`

Change file and directory permissions.

`chown`, `chgrp`

Change ownership and group assignment.

`umask`

Define default permissions for new files and directories.

Processes and services

Monitoring and controlling running systems

`ps`, `ps aux`

Inspect running processes.

`top`, `htop`

View live CPU, memory, and process activity.

`kill`, `pkill`, `killall`

Terminate processes by PID or name.

`systemctl`

Start, stop, restart, enable, and inspect services on systemd systems.

`service`

Manage services on older distributions or compatibility layers.

`journalctl`

View systemd logs and service logs.

`uptime`

Check load average and system uptime.

`free -h`

Inspect memory usage quickly.

Networking

Connectivity, addressing, and remote access commands

`ip a`, `ip r`

Inspect IP addresses and routing tables.

`ping`

Test basic connectivity to hosts.

`traceroute` or `tracepath`

Trace the path packets take across networks.

`nslookup`, `dig`, `host`

Query DNS and inspect name resolution.

`ss`, `netstat`

Inspect listening ports and active connections.

`curl`, `wget`

Test web endpoints and retrieve remote files.

`ssh`

Remote login and command execution.

`scp`, `rsync`

Secure file transfer and synchronization.

`iptables`, `nft`

Inspect and manage firewall rules depending on the distribution.

Storage and logs

Disk usage, filesystems, and logging commands

`df -h`

Check mounted filesystems and free disk space.

`du -sh`

Measure directory and file usage.

`lsblk`

List block devices and partitions.

`mount`, `umount`

Mount and unmount filesystems.

`fdisk`, `parted`

Inspect and manage disk partition layouts.

`tar`

Archive and extract files for backup and transfer.

`gzip`, `gunzip`, `zip`, `unzip`

Compress and decompress files.

`tail -f`

Watch logs in real time during troubleshooting.

`dmesg`

Read kernel messages for hardware and boot issues.
