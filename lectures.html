Learning outcomes

What students should leave with

  • Explain systems administration using systems theory and operating system concepts.
  • Explain how distributed systems and virtualization enable cloud computing.
  • Evaluate IAM and governance choices using least-privilege and secure access principles.
  • Evaluate VPC, subnetting, routing, and cloud-network security designs against networking principles.
  • Explain how EC2 and Infrastructure as Code support repeatable compute provisioning.
  • Evaluate Linux administration practices for cloud-hosted enterprise systems using SSH and AWS Systems Manager.
  • Evaluate storage strategies using performance, availability, cost, and AWS storage architecture principles.
  • Evaluate database solutions using application fit, consistency, scalability, availability, and AWS managed database services.
  • Analyze production-ready web application architectures using availability, scalability, security, and AWS delivery services.
  • Recommend monitoring and observability strategies using metrics, logs, auditing, and AWS operational services.
  • Evaluate layered cloud security practices using preventive, detective, and corrective controls with AWS security services.
  • Evaluate resilient infrastructure designs using availability, scalability, fault tolerance, and reliability engineering principles.
  • Evaluate business continuity and disaster recovery strategies using backup design, RPO, RTO, and AWS data protection services.
