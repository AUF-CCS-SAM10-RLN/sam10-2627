Course rule

What students are expected to produce

Not every document a professional systems administrator might eventually maintain should be a separate graded submission in this course. To keep the workload realistic, each lab produces one main artifact only. Some professional document types are therefore embedded inside broader course artifacts instead of being required as separate standalone submissions.

Foundation and design

Topics 1 to 5

Early-course artifacts focused on system understanding, cloud models, access design, networking, and compute planning.

Topic 1: Infrastructure overview document

  • Main purpose: document the environment, major components, ownership, and shared-responsibility boundaries
  • Closest professional artifact: system overview and starter inventory

Topic 2: Deployment recommendation memo

  • Main purpose: justify deployment and virtualization choices with cost-aware reasoning
  • Closest professional artifact: architecture decision memo

Topic 3: Access control and governance policy note

  • Main purpose: define roles, permissions, MFA expectations, and governance rules
  • Closest professional artifact: access control matrix with policy notes

Topic 4: Network design brief

  • Main purpose: explain VPC layout, subnet roles, route intent, and traffic controls
  • Closest professional artifact: logical network design document

Topic 5: Compute provisioning runbook

  • Main purpose: capture provisioning steps, instance decisions, and introductory IaC planning
  • Closest professional artifact: provisioning SOP or server build runbook

Platform and service delivery

Topics 6 to 10

Mid-course artifacts focused on Linux administration, storage, databases, application delivery, and monitoring.

Topic 6: Linux administration quick guide

  • Main purpose: summarize shell, file-system, permission, process, package, and remote-access checks
  • Closest professional artifact: Linux administration checklist

Topic 7: Storage strategy brief

  • Main purpose: document storage choices based on architecture, performance, and lifecycle needs
  • Closest professional artifact: storage planning note

Topic 8: Database strategy brief

  • Main purpose: justify relational or NoSQL choices, backup approach, and managed-service fit
  • Closest professional artifact: database design recommendation

Topic 9: Application deployment architecture brief

  • Main purpose: explain multi-tier deployment, load balancing, DNS, TLS, and application flow
  • Closest professional artifact: application deployment design document

Topic 10: Monitoring and observability strategy brief

  • Main purpose: define priority metrics, log visibility, alerts, and operational monitoring choices
  • Closest professional artifact: monitoring plan

Operations and optimization

Topics 11 to 15

Late-course artifacts focused on security, resilience, recovery, maintenance, and optimization.

Topic 11: Cloud security strategy brief

  • Main purpose: explain layered security controls, risks, and AWS security-service use
  • Closest professional artifact: security control summary and policy support note

Topic 12: Reliability and scaling strategy brief

  • Main purpose: document availability, scaling, fault-tolerance, and health-check decisions
  • Closest professional artifact: reliability design note

Topic 13: Business continuity and disaster recovery strategy brief

  • Main purpose: define backup scope, RPO/RTO thinking, recovery choices, and continuity planning
  • Closest professional artifact: backup and recovery plan

Topic 14: Operations maintenance and service management brief

  • Main purpose: document maintenance approach, patching, change handling, automation, and service expectations
  • Closest professional artifact: operations plan combining SOP and runbook elements

Topic 15: Architecture optimization and Well-Architected review brief

  • Main purpose: assess architecture quality and prioritize improvements across the AWS Well-Architected pillars
  • Closest professional artifact: architecture review report
