Topic 1

Foundations of Systems Administration

This lecture establishes the basic language of systems administration before students move into AWS infrastructure, Linux administration, and operational maintenance.

Lecture focus

  • Understand what systems administration is.
  • Connect infrastructure, operating systems, and cloud platforms.
  • Explain why AWS changes, but does not remove, administrative responsibility.

Subtopics

  • Systems Theory
  • Computer Systems and IT Infrastructure
  • Operating Systems Fundamentals
  • Client-Server Architecture
  • Roles and Responsibilities of Systems Administrators
  • AWS Global Infrastructure
  • AWS Shared Responsibility Model

Objectives

What students should be able to explain

  • Explain the principles, functions, and responsibilities of systems administration based on systems theory and operating system concepts.
  • Differentiate traditional on-premises infrastructure from cloud-based infrastructure using AWS Global Infrastructure.
  • Describe how AWS Regions and Availability Zones support distributed and resilient computing.
  • Explain the AWS Shared Responsibility Model and its implications for cloud administration.

Chapter insight

What exactly does a systems administrator do?

Core idea

The role is broad. A systems administrator manages and maintains computer and network systems for an organization, but the exact duties vary by environment, team size, and operational needs.

Typical responsibilities

  • Provision and connect systems
  • Manage users, software, storage, and backups
  • Troubleshoot failures and restore service
  • Write scripts, automate work, and document procedures

Subtopic 1

Systems Theory

Main idea

A system is a set of connected parts working together to achieve a purpose. Systems administration views infrastructure as an interdependent whole rather than isolated devices.

Key points

  • Inputs, processes, outputs, and feedback matter.
  • One failure can affect other connected components.
  • Administration requires understanding relationships, not just individual tools.

Simple systems view

Input User requests
→
Process System services
→
Output Delivered service
↺
Feedback Logs, alerts, user reports

Subtopic 2

Computer Systems and IT Infrastructure

Core components

  • Compute
  • Storage
  • Networking
  • Users and access
  • Applications and services

Administration view

Infrastructure is the operational foundation of IT services. Systems administrators keep it available, secure, documented, and maintainable.

Subtopic 3

Operating Systems Fundamentals

What an OS does

  • Manages hardware resources
  • Runs processes and services
  • Handles files, memory, and users
  • Provides an interface for administration

Why it matters

Every server or cloud instance depends on an operating system layer. Good administration requires understanding services, permissions, updates, and failures at this level.

Subtopic 4

Client-Server Architecture

Basic model

  • Clients request services.
  • Servers process requests and return responses.
  • Networks carry traffic between them.

Administrative importance

Systems administrators must know where requests start, where services run, and which dependencies must remain available for users to succeed.

Subtopic 5

Roles and Responsibilities of Systems Administrators

Typical responsibilities

  • Provision systems
  • Manage users and permissions
  • Apply updates and maintenance
  • Monitor health and troubleshoot issues
  • Protect systems through security controls
  • Document environments and procedures

Professional view

Administration is both technical and operational. The role is not just to build systems, but to keep them reliable over time.

Chapter insight

System administration as a profession and field of study

Important points

  • The field is real and demanding, even if job titles vary widely.
  • Experience matters, but experience alone is not enough.
  • Formal theory and practical work strengthen each other.
  • Students need hands-on access to real systems to learn effectively.

Why this matters in SAM10

This course should not train students to memorize commands only. It should build underlying principles, judgment, and practical operational habits.

Subtopic 6

AWS Global Infrastructure

Key terms

  • Region
  • Availability Zone
  • Edge location

Why it matters

AWS distributes infrastructure across multiple physical locations. This supports resilience, lower latency choices, and workload distribution across failures.

Region and Availability Zones

Region One geographic area containing multiple isolated Availability Zones
AZ 1 Workloads can run here
AZ 2 Workloads can fail over here

Subtopic 7

AWS Shared Responsibility Model

AWS is responsible for

  • Security of the cloud
  • Physical facilities
  • Core infrastructure and managed platform layers

Customer is responsible for

  • Security in the cloud
  • Identity and access configuration
  • Operating systems, data, and workload settings where applicable

Responsibility split

SAM10 on AWS side Facilities, hardware, and core cloud platform protection
Customer side Accounts, data, OS settings, and workload configuration

Recap

Key takeaways and short check

Recap

  • Systems administration is about managing connected systems, not isolated parts.
  • Operating systems and client-server design remain central in cloud computing.
  • AWS changes infrastructure delivery, but not the need for administrative responsibility.

Short check

  1. What is one example of feedback in a system?
  2. Why do Regions and Availability Zones matter?
  3. Give one responsibility that stays with the customer in AWS.
