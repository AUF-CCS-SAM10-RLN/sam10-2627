Topic 1

Foundations of Systems Administration Lab

Build and analyze a basic cloud-based system using core infrastructure and AWS concepts.

3 hours

Goal

Apply systems administration concepts by identifying system components, mapping client-server relationships, and examining how AWS infrastructure and the Shared Responsibility Model affect administration.

Activities

  1. Review systems theory, operating systems, client-server architecture, and administrator roles.
  2. Draw a simple on-premises IT environment and identify key components and responsibilities.
  3. Redraw the same environment as a cloud-based design using AWS Regions and Availability Zones.
  4. Explore the AWS Management Console or a guided simulation to inspect infrastructure concepts.
  5. Complete Shared Responsibility Model scenarios and classify AWS versus customer duties.
  6. Submit a short reflection comparing on-premises and cloud administration.

Expected outputs

  • A labeled infrastructure diagram
  • An on-premises versus cloud comparison table
  • Shared responsibility scenario answers
  • A short written reflection

Skills reinforced

  • Systems thinking
  • Client-server analysis
  • Infrastructure comparison
  • AWS foundational literacy
  • Administrative responsibility boundaries

Topic 2

Cloud Computing and Virtualization Fundamentals Lab

Compare cloud deployment models, inspect virtualization approaches, and estimate basic AWS compute costs.

3 hours

Goal

Connect distributed systems and virtualization theory to practical infrastructure choices, then evaluate AWS compute and pricing decisions for a small organization.

Activities

  1. Review cloud deployment models, service models, distributed systems, and virtualization concepts.
  2. Classify scenarios as public, private, hybrid, or multi-cloud and justify each choice.
  3. Compare hypervisors, virtual machines, and containers using a feature matrix.
  4. Inspect AWS compute options such as EC2 and container-based services through guided notes or screenshots.
  5. Match AWS services to IaaS, PaaS, and SaaS-style use cases.
  6. Estimate costs for a simple deployment using AWS pricing examples and discuss planning trade-offs.

Expected outputs

  • A deployment model classification worksheet
  • A virtual machine versus container comparison table
  • An AWS service model mapping activity
  • A short pricing and infrastructure recommendation

Skills reinforced

  • Cloud model analysis
  • Virtualization literacy
  • Service model differentiation
  • AWS compute awareness
  • Cost-conscious infrastructure planning
